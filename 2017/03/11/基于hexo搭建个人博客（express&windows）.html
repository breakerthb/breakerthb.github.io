基于hexo搭建个人博客（express&windows）

前言

记录这一次搭建hexo静态博客的过程以及遇到的一些问题。

1. Ubuntu安装

Ubuntu安装包括网卡配置，代理配置，源配置等等，请查看Ubuntu的安装。

2. hexo安装和搭建和部署

hexo博客搭建的基础流程为：安装Node.js，安装npm，安装主题（Next），本地测试运行，部署。下面分别介绍：

  • 安装nodejs

    • 用nvm安装（因为要从github上下载，所以还没成功过）

    • windows下安装包，直接从nodejs官网下载安装包，在windows下直接双击msi安装， 默认情况下Node.js安装完毕，会自动在系统的path环境变量中配置了node.exe的目录路径，但是可能你安装完成后，dos命令下输入node提示错误。打开系统环境变量，发现确实已经配置了，但是dos下运行“set path”又看不到nodejs的配置，那就是人品的问题了。。重启电脑重新加载就可以了。也可以删掉自动配置的，手动再加上即可。

    • linux下安装包（全局安装）

      解压到/home/xiaojie/nodejs/node-v6.2.0-linux-x64/

配置环境变量，vim /etc/profile，添加下面两句：

  export NODE_HOME=/home/xiaojie/nodejs/node-v6.2.0-linux-x64
  export PATH=$NODE_HOME/bin:$PATH

编译/etc/profile 使配置生效 ，source /etc/profile

检查是否安装成功，node -v

  • 安装npm

    • 新版本的nodejs已经集成了npm，所以可以通过输入”npm -v”查看npm是否安装成功。如果没有安装的话，可以通过sudo apt-get install npm 来全局安装npm。

    • 由于公司安保的问题，需要先申请web访问例外，然后配置源和代理，才能正常使用。
      首先几个重要的配置项一一介绍

      prefix   -- 全局安装的路径，也就是npm install -g 安装的模块在哪个位置。这个看个人喜好。我喜好将他设置到安装路径下面。

proxy -- 代理（http的代理是用这个），代理连外网的朋友这个就需要配置了。

https_proxy -- https代理

registry -- 类似linux的软件源，这个一定要修改的 

    #npm config set prefix "c:\nodejs"
    npm config set proxy=http://109.105.113.200:8080
    npm config set https_proxy=http://109.105.113.200:8080
    npm config set registry=https://registry.npm.taobao.org/

  • Hexo的搭建

    • Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。 安装Hexo的非常简单，只要一条命令，前提是安装好Node.js,

      npm install -g hexo-cli

    • 同样配置环境变量，vim /etc/profile，添加下面两句：

      export HEXO_HOME=/home/xiaojie/nodejs/Website1/node_modules/hexo-cli

export PATH=$HEXO_HOME/bin:$PATH

      编译/etc/profile 使配置生效 ，source /etc/profile

      此时就可以在全局作用域下，hexo -v检查是否能正常使用。

- 本地启动Hello World与Hexo简单使用

    + 新建一个文件夹，取名为blog，cd到该文件夹，安装必要的文件

            hexo init

            npm install

    + 有了必要的各种配置文件之后就可以在本地预览效果了

            hexo g #生成静态文件

            hexo s #在本地服务器上运行

    + 之后打开浏览器并输入IP地址 http://localhost:4000/

  • 安装Next主题

    • Hexo 安装主题的方式非常简单，只需要将主题文件拷贝至站点目录的 themes 目录下， 然后修改下配置文件即可，公司不能使用git下载，所以我在github上下载下来，链接在这里

    • 启用主题

      • 打开站点配置文件，找到theme字段，并将其值修改为next

        theme：next

        然后hexo s即可以预览效果

      • 更换主题外观

        Next有三个主题外观，默认使用的Muse，直接更改主题配置文件的scheme参数即可，如果想修改站点的字体，可以通过站点配置文件中的，language：zh-CN来修改，再执行hexo clean 和 hexo s可预览效果。

  • 部署

    本文介绍部署在nodejs的express服务器上，操作系统windows。

    • 安装express应用生成器 

      npm install express-generator -g

      创建应用

      express myblog

      安装依赖包

      npm install

  + 在blog目录下，执行<font color=#0099ff>hexo g</font>构建静态目录public，用于部署

+ 将public文件夹替换掉/myblog/public文件夹

+ 将app.js里的中间件替换掉


         /* GET home page. */
        app.get('/', function(req, res, next) {
          var options = {
            root: __dirname + '/public/',
            dotfiles: 'deny',
            headers: {
                'x-timestamp': Date.now(),
                'x-sent': true
            }
          };
          var fileName = 'views/index.html';
          res.sendFile(fileName, options, function (err) {
            if (err) {
              console.log(err);
              res.status(err.status).end();
            }
            else {
              console.log('Sent:', fileName);
            }
          });
        });

+ 启动node app.js,查看http://109.105.112.134：3000

+ windows创建服务，创建service.js 内容为：

    var Service = require('node-windows').Service;

    // Create a new service object
    var svc = new Service({
    name:'SharingBlog',
    description: 'a blog for sharing knowledge',
    script:__dirname+'/bin/www'
    });
    console.log(svc.script);
    // Listen for the "install" event, which indicates the
    // process is available as a service.
    svc.on('install',function(){
    console.log('Install complete.');
    //svc.start();
    });

    //svc.user.account = 'SYSTEM';
    svc.install();

以及uninstall.js，内容为：

    var Service = require('node-windows').Service;

    // Create a new service object
    var svc = new Service({
    name:'SharingBlog',
    script: __dirname+'/bin/www'
    });

    // Listen for the "uninstall" event so we know when it's done.
    svc.on('uninstall',function(){
    console.log('Uninstall complete.');
    });

    // Uninstall the service.
    svc.uninstall();

+ 制作脚本

        "%~dp0SamsungDCMServer.exe" "%~dp0Admin\service.js"
        net start SharingBlog

    双击脚本安装windos服务，访问http://localhost：3000，测试网站是否部署成功。
